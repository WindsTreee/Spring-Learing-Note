LinkedHashmap就是LinkedList和Hashmap的结合，因为Hashmap访问时不是按照元素放入的顺序访问的。它是非线程安全的。 参考链接

LinkedHashmap和LinkedList都是双向链表

LinkedHashmap可以用于实现LRU缓存(未实践)


Hashmap的keySet方法可以返回hashmap中所有的key值，返回类型是Set<键的类型>，所以可以使用keySet.iterator()进行遍历访问


迭代器的.next()方法返回的是当前迭代的元素，并把迭代的游标向后移，所以要访问具体的值要使用.next方法

在迭代器迭代过程中不能对集合进行写删除等会改变元素值或结构的操作，否则会报ConcurrentModificationException错误

迭代器Iterator是一个接口，可以看作是所以迭代对象的母接口，具体如何迭代由它实现

Iterator i=map.keySet().iterator();
while (i.hasNext()){
    System.out.println(i.next());;
}


Iterable也是一个接口，实现了这个接口的集合对象支持迭代，实现了这个接口的就可以用foreach访问集合元素。

一般都是实现Iterable接口而不实现Iterator接口，很多类实现了Iterable接口之后可以调用iterator方法。 参考链接

Hashmap的foreach遍历：

map.forEach((k,v)->{
    System.out.println(v);
});

普通集合的foreach：

        for (String string : list) {
            System.out.println(string);
        }
